VBA Best Practices Guide

Coding standards, error handling, and performance tips for professional VBA development

Write VBA code that's maintainable, reliable, and fast.

This guide covers essential best practices I've learned from 15+ years of VBA development. Follow these standards to create code that works today and can be maintained tomorrow.

Coding Standards

1. Use Option Explicit

Always start modules with Option Explicit. This forces variable declaration and catches typos at compile time.

Option Explicit Sub ProcessData() Dim ws As Worksheet Dim i As Long ' Your code here End Sub
2. Declare Variable Types Explicitly

Don't use Variant unless necessary. Explicit types improve performance and catch errors early.

' Good Dim rowCount As Long Dim fileName As String Dim isComplete As Boolean ' Bad Dim rowCount Dim fileName Dim isComplete
3. Use Meaningful Variable Names

Names should describe purpose, not type. Use camelCase or PascalCase consistently.

' Good Dim customerCount As Long Dim lastRow As Long Dim isDataValid As Boolean ' Bad Dim x As Long Dim temp As String Dim flag As Boolean
4. Avoid Magic Numbers

Use named constants instead of hardcoded values. Makes code self-documenting.

' Good Const MAX_ROWS As Long = 10000 Const HEADER_ROW As Long = 1 If rowCount > MAX_ROWS Then ' Handle error End If ' Bad If rowCount > 10000 Then ' What does 10000 mean? End If
5. Keep Functions Small and Focused

Each function should do one thing well. Break complex functions into smaller, reusable pieces.

' Good - Single responsibility Function GetLastRow(ws As Worksheet) As Long GetLastRow = ws.Cells(ws.Rows.Count, 1).End(xlUp).Row End Function ' Bad - Does too much Sub DoEverything() ' 200 lines of mixed logic End Sub

Error Handling

6. Always Use Error Handling

Every procedure that can fail should have error handling. Use On Error GoTo pattern.

Sub ProcessFile() On Error GoTo ErrorHandler Dim ws As Worksheet Set ws = ThisWorkbook.Worksheets("Data") ' Your code here Exit Sub ErrorHandler: MsgBox "Error: " & Err.Description, vbCritical ' Clean up resources Set ws = Nothing End Sub
7. Clean Up Resources in Error Handler

Close files, release objects, and reset settings even when errors occur.

Sub ProcessData() Dim wb As Workbook On Error GoTo ErrorHandler Set wb = Workbooks.Open("C:\Data\file.xlsx") ' Process data wb.Close SaveChanges:=False Exit Sub ErrorHandler: If Not wb Is Nothing Then wb.Close SaveChanges:=False End If MsgBox "Error processing file: " & Err.Description End Sub
8. Log Errors for Debugging

Record errors to a log file or worksheet for troubleshooting.

Sub LogError(errorMsg As String) Dim logFile As String logFile = ThisWorkbook.Path & "\error_log.txt" Open logFile For Append As #1 Print #1, Now & " - " & errorMsg Close #1 End Sub

Performance Tips

9. Turn Off Screen Updating

Disable screen updates during long operations. Dramatically improves speed.

Sub ProcessLargeDataset() Application.ScreenUpdating = False Application.Calculation = xlCalculationManual ' Your code here Application.Calculation = xlCalculationAutomatic Application.ScreenUpdating = True End Sub
10. Use Arrays Instead of Cell-by-Cell Operations

Read/write data in bulk using arrays. 10-100x faster than looping through cells.

' Good - Fast Dim dataArray As Variant dataArray = ws.Range("A1:C1000").Value ' Process array ws.Range("A1:C1000").Value = dataArray ' Bad - Slow Dim i As Long For i = 1 To 1000 ws.Cells(i, 1).Value = ProcessValue(ws.Cells(i, 1).Value) Next i
11. Avoid Select and Activate

Work with objects directly. Select/Activate is slow and error-prone.

' Good ws.Range("A1").Value = "Hello" ' Bad ws.Select Range("A1").Select ActiveCell.Value = "Hello"
12. Use With Statements

Reduces object references and improves readability.

' Good With ws.Range("A1:C10") .Font.Bold = True .Interior.Color = RGB(255, 0, 0) .NumberFormat = "#,##0" End With ' Bad ws.Range("A1:C10").Font.Bold = True ws.Range("A1:C10").Interior.Color = RGB(255, 0, 0) ws.Range("A1:C10").NumberFormat = "#,##0"

Code Organization

13. Use Modules for Reusable Code

Put reusable functions in standard modules. Keep worksheet-specific code in worksheet modules.

14. Add Comments for Complex Logic

Explain "why," not "what." Code should be self-documenting, but complex business logic needs comments.

' Calculate weighted average using fiscal year weights ' Weights are based on days in each quarter, not equal distribution Dim weightedAvg As Double weightedAvg = (Q1Value * Q1Days / TotalDays) + (Q2Value * Q2Days / TotalDays)
15. Version Control Your Code

Export modules to text files and use version control (Git). Makes collaboration and rollback possible.

Security & Reliability

16. Validate Input Data

Never trust user input or external data. Validate before processing.

Function IsValidEmail(email As String) As Boolean If InStr(email, "@") = 0 Then IsValidEmail = False Exit Function End If ' More validation... IsValidEmail = True End Function
17. Use Early Binding When Possible

Early binding (with references) is faster and provides IntelliSense. Use late binding only for compatibility.

18. Test with Real Data

Test with actual data volumes and edge cases. Small test files hide performance problems.
